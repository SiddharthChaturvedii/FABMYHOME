FABMYHOME - Developer Guide & Project Showcase
===========================================

Welcome to the FABMYHOME technical documentation. This document serves as both a developer onboarding guide and a "Showcase Dossier" detailing the engineering challenges we solved from concept to production.

---

1. ARCHITECTURE & STRUCTURE
---------------------------
Next.js 15/16 (App Router), React, TypeScript, Tailwind CSS, Zustand, and Framer Motion.

- /app : Core routing and global styles (globals.css).
- /components : Feature-based modules (Hero, Quiz, Complete-The-Look, Visual-Search).
- /store : Zustand state managers (uiStore.ts for coordination, quizStore.ts for onboarding).
- /lib/utils.ts : The cn() helper for class merging (Tailwind-merge + Clsx).

---

2. ENGINEERING CHALLENGES & DEBUG HISTORY (The "Bigger Ones")
-------------------------------------------------------------

A) [EARLY PHASE] The Hydration & Coordinate Conflict
- Problem: The complex Hero sequence caused "Hydration Mismatch" errors because the client-side Framer Motion triggers didn't match the initial server-rendered HTML.
- Debug/Fix: Centralized the entire entrance sequence into a Zustand store (uiStore). We implemented a "mounted" state guard and coordinated the LoadingCurtain with the introStage (0-3). This ensured the browser only started animations once the React tree was stable.

B) [MIDDLE PHASE] The 50MB Performance Wall
- Problem: Our high-fidelity hero animation originally relied on 192 static image frames. This caused extreme TTI (Time-To-Interactive) delays, especially on 4G mobile networks.
- Debug/Fix: We refactored the entire system from frame-by-frame rendering to hardware-accelerated video streaming. We built a responsive "HeroVideo" component that detects viewport orientation to serve optimized .mp4 files (laptop vs mobile), cutting the asset weight by 90% while improving visual quality.

C) [MIDDLE PHASE] Layout Overlaps & Z-Index Layering
- Problem: As we added Glassmorphic Mega Menus, the hero video would often "leak" through or navigation items would become unclickable on mobile.
- Debug/Fix: We implemented a strict z-index hierarchy and added 'pointer-events-none' to background video containers. We also solved a "Ref Forwarding" issue where the Radix UI HoverCard trigger wasn't firing because it was wrapped in a custom component that didn't expose its DOM reference.

D) [RECENT] The Vercel PostCSS Stabilization
- Problem: During deployment, Vercel threw "Unknown Utility Class" errors for modern Tailwind v4 features (like backdrop-blur-2xl).
- Debug/Fix: We had to debug the PostCSS pipeline and move complex effects (Glassmorphism, Marquee animations) into pure CSS variables and keyframes within globals.css to ensure build-time stability across different environments.

E) [NOW] The Mobile Autoplay & Interaction Lock
- Problem: Mobile browsers were showing a "Play Button" on the background video and pausing it by default to save data.
- Debug/Fix: We implemented a "Triple-Lock" system:
  1. Forced muted/playsinline attributes.
  2. Manual .play() triggers in a useEffect hook.
  3. A transparent 'pointer-events-none' overlay to prevent accidental user pauses.

3. DESIGN SYSTEM & TYPOGRAPHY
-----------------------------
- Primary Font: Inter (Variable weight). Chosen for its "Institutional Apple" feel.
- Heading Style: All-Caps, Font-Black (900), Tracking-Tighter.
- Signature Color: Lighter Brand Teal (#124444) for Curtain/Footer/Testimonials.
- Interactive Physics: Custom 35-degree 3D TiltCards with GSAP-powered ScrollFloat reveals.
